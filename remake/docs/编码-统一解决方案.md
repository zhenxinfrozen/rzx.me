# 中文编码统一解决方案

> 结论：以当前方案为定案。终端层已强制 UTF-8，后续如再出现乱码再专项处理。

## 1. 当前基准（本项目）
- PowerShell 启动即强制 UTF-8。
- VS Code 默认 UTF-8（若已配置 workspace settings 则以其为准）。
- 项目文件统一使用 UTF-8（无 BOM），换行符使用 LF。

## 2. 必做配置

### 2.1 PowerShell 全局 UTF-8（已启用）
写入 PowerShell Profile：
```powershell
$OutputEncoding = [System.Text.Encoding]::UTF8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
[Console]::InputEncoding = [System.Text.Encoding]::UTF8
$PSDefaultParameterValues['Out-File:Encoding'] = 'utf8'
$PSDefaultParameterValues['*:Encoding'] = 'utf8'
chcp 65001 | Out-Null
```

### 2.2 VS Code 工作区（如需确认）
建议保持：
- `files.encoding = utf8`
- `files.autoGuessEncoding = true`
- `files.eol = "\n"`
- 终端启动时强制 UTF-8

### 2.3 VS Code 全局配置（可选）
如需所有项目统一生效，可在用户设置中保持与工作区相同的编码配置。优先级为：
1) 工作区设置 `.vscode/settings.json`
2) 用户设置 `%APPDATA%\Code\User\settings.json`
3) VS Code 默认设置

### 2.4 EditorConfig（推荐保留）
```ini
[*]
charset = utf-8
end_of_line = lf
insert_final_newline = true
trim_trailing_whitespace = true
```

### 2.5 Git 相关（如使用 Git）
```bash
git config --global core.quotepath false
git config --global gui.encoding utf-8
git config --global i18n.commit.encoding utf-8
git config --global i18n.logoutputencoding utf-8
git config --global core.autocrlf false
git config --global core.eol lf
```

## 3. 验证要点
```powershell
[Console]::OutputEncoding.CodePage
[Console]::InputEncoding.CodePage
$OutputEncoding.CodePage
chcp
Write-Output "测试中文: 你好世界"
```
期望：代码页为 65001，中文输出正常。

## 4. 发生乱码时的处理策略
1. **先确认文件原始是否正确**：从正确来源恢复（服务器、远端、历史版本）。
2. **不要尝试“逆向修复乱码文本”**，乱码通常不可逆。
3. 可用工具检测：
   - `php tools/check-encoding.php`
   - `php tools/check-encoding.php --fix`（谨慎）

## 5. AI/脚本读写文件的正确方式
```powershell
# 推荐
[System.IO.File]::ReadAllText($path, [System.Text.Encoding]::UTF8)
[System.IO.File]::WriteAllText($path, $content, [System.Text.Encoding]::UTF8)

# 如需 Get-Content/Set-Content
Get-Content $path -Encoding UTF8
Set-Content $path -Value $content -Encoding UTF8
```

## 6. 最终说明
当前方案已经满足日常使用。后续如再出现乱码，再定位具体场景处理即可。
