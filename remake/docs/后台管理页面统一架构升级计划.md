## 一、升级概述

### 1.1 核心目标
- ✅ 统一UI/UX设计语言
- ✅ 组件化CSS/JS架构
- ✅ 三栏布局标准化
- ✅ 为未来MySQL迁移做准备
- ✅ 提高代码可维护性

### 1.2 设计原则
1. **组件优先**: 90%样式来自组件，10%页面特定样式
2. **不要重复造轮子**: 不覆盖组件已有的样式（如`.admin-list-item`）
3. **统一数据流**: 标准化CRUD API（为MySQL迁移准备）
4. **逐步迁移**: 创建`-new.php`版本，保留旧版本作为备份

## 三、升级步骤标准化

### 3.1 每个页面的升级流程

#### 阶段1: 准备工作 (30分钟)
1. **分析旧版页面**
   - [ ] 读取旧版文件，了解功能点
   - [ ] 列出所有CRUD操作
   - [ ] 识别特有UI元素（如Comics的双图标）

2. **确认后端支持**
   - [ ] 检查 `app/Admin/Controllers/` 中的控制器
   - [ ] 确认数据文件位置（`app/storage/data/`）
   - [ ] 验证API路由是否存在

#### 阶段2: 创建基础结构 (1-2小时)
1. **复制模板**
   ```bash

   cp app/Admin/Views/pages/admin-sketchbook-new.php app/Admin/Views/pages/admin-xxx-new.php
   ```

2. **修改基础信息**
   - [ ] 更新页面标题和元数据

3. **配置路由**
   - [ ] 在 `app/Admin/Controllers/AdminIndexController.php` 添加路由
   - [ ] 测试访问 `/admin?page=xxx-new`

#### 阶段3: 实现功能 (2-3小时)

   - [ ] 实现数据加载函数 `loadXxxList()`

## 七、风险与注意事项

### 7.1 常见坑点
1. **数据结构差异**: 
   - Comics使用关联数组 (`comic_ID` => data)
   - Sketchbook/Galleries使用索引数组 with `id` field
   - 需要正确处理 `foreach` 和 `array_values()`

2. **CSS覆盖问题**:
   - ❌ 不要添加自定义 `.admin-list-item` 样式
   - ❌ 不要覆盖组件已有的hover/active效果
   - ✅ 只添加页面特有UI元素样式

3. **路由配置**:
   - 记得在 `AdminIndexController.php` 添加新路由
   - 使用 `?page=xxx-new` 访问新版本

4. **浏览器缓存**:
   - CSS/JS改动后更新版本号 `?v=2.1`
   - 强制刷新 `Ctrl + F5`

### 7.2 回滚策略
- 保留所有旧版文件（`admin-xxx.php`）
- 新版本出问题时，修改路由即可回滚
- 数据文件有备份机制
