# Videos 页面自动同步功能说明

## ✅ 已修复的问题

### 1. **文件删除自动同步** 
现在系统会在每次刷新页面时自动检测：
- ✅ **新增的视频文件** - 自动添加到对应分组
- ✅ **删除的视频文件** - 自动从数据库中移除
- ✅ **新增的分组目录** - 自动创建新分组
- ✅ **删除的分组目录** - 自动移除整个分组

**工作原理**：
```php
sync_videos_with_directory($existingGroups)
├── 第一步：删除不存在的文件和分组
│   ├── 检查JSON中的每个分组是否还存在于文件系统
│   ├── 检查每个视频文件是否还存在
│   └── 删除不存在的条目
└── 第二步：添加新文件和分组
    ├── 扫描文件系统中的新分组
    ├── 扫描每个分组中的新视频文件
    └── 添加新条目到JSON
```

### 2. **HTML5提示位置修正**
现在HTML5提示正确显示在**每个分组视频列表的底部**，而不是整个页面底部。

**修改点**：
- JavaScript中的HTML5提示生成位置
- 每次点击分组时，HTML5提示都会随该分组的视频一起加载
- 切换到其他分组时，会显示新的HTML5提示

## 📖 使用指南

### 添加新视频
1. 将视频文件（MP4/WebM等）放入对应分组目录
   ```
   public/assets/videos/video-gallery/
   ├── animation-clips/
   │   └── 新视频.mp4       ← 放这里
   ├── comic-anim/
   └── storyboard/
   ```

2. **刷新浏览器页面**即可看到新视频（无需手动扫描）

### 删除视频
1. 直接从文件系统删除视频文件
2. **刷新浏览器页面**，系统会自动清理数据库

### 创建新分组
1. 在 `public/assets/videos/video-gallery/` 下创建新文件夹
2. 将视频文件放入新文件夹
3. **刷新浏览器页面**，新分组自动出现

### 删除分组
1. 删除整个分组文件夹
2. **刷新浏览器页面**，分组自动从菜单中消失

## 🔧 技术细节

### 同步策略
- **增量同步**：默认模式，只检查变化的文件（快速）
- **完全重建**：使用 `--force` 参数完全从目录重建（慢但彻底）

### 命令行工具
```bash
# 强制完全重新扫描
php app/Console/ScanVideos.php --force

# 或者通过PHP函数
get_videos_with_auto_scan($forceRescan = true)
```

### 性能优化
- 使用路径哈希表进行快速查找（O(1)复杂度）
- 只在检测到变化时才写入JSON文件
- 避免重复扫描，利用缓存数据

## 📝 注意事项

1. **预览图命名**：视频预览图应与视频同名（扩展名为 .jpg/.png/.webp）
2. **支持格式**：MP4, WebM, MOV, AVI, OGV
3. **自动保存**：检测到变化时会自动更新 `app/storage/data/video_data.json`
4. **权限要求**：确保PHP对 `app/storage/data/` 目录有写入权限
