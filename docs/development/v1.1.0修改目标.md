# v1.1.0 目录结构标准化方案

## 📋 项目结构问题分析

### 当前问题分析：

#### 1. **数据文件存放混乱** ❌
```
storage/comic_data.json                    # 根目录storage
app/storage/data/video_data.json          # app目录storage
app/Data/custom_thumbnail_presets.json    # app/Data目录
app/Data/single_works_image_order.json
app/Data/sketchbook_image_order.json
app/Data/thumbnail_configs.json
```

#### 2. **配置与数据混合** ❌
```
app/Config/ (PHP文件):
├── app.php, database.php, routes.php     # 系统配置
├── single_works_config.php              # 业务配置
├── sketchbook_config.php
├── video_gallery_sort.php               # 排序配置
└── video_thumbnail_overrides.php

app/Data/ (JSON文件):
├── custom_thumbnail_presets.json        # 缩略图预设
├── thumbnail_configs.json               # 缩略图配置
└── *_image_order.json                   # 图片排序
```

#### 3. **存储层级不一致** ❌
```
根目录: storage/
app目录: storage/ + Data/
```

## 🎯 标准化目录结构方案

### 推荐的新结构：
```
app/
├── Config/           # 所有配置 (PHP文件)
│   ├── system/       # 系统配置
│   │   ├── app.php
│   │   ├── database.php
│   │   └── routes.php
│   └── modules/      # 业务模块配置
│       ├── comics.php
│       ├── single-works.php
│       ├── sketchbook.php
│       └── video-gallery.php
├── Storage/          # 所有数据存储 (JSON文件)
│   ├── data/         # 业务数据
│   │   ├── comics.json
│   │   ├── single-works.json
│   │   ├── sketchbook.json
│   │   └── video-gallery.json
│   ├── config/       # 运行时配置
│   │   ├── thumbnail-presets.json
│   │   ├── image-orders.json
│   │   └── sort-orders.json
│   ├── cache/        # 缓存文件
│   └── logs/         # 日志文件
└── Models/           # 数据访问层
    ├── ComicsModel.php
    ├── SingleWorksModel.php
    ├── SketchbookModel.php
    └── VideoGalleryModel.php
```

### 文件迁移映射：
```
# 当前位置                          → 新位置
────────────────────────────────────────────────────────────────
storage/comic_data.json             → app/Storage/data/comics.json
app/storage/data/video_data.json    → app/Storage/data/video-gallery.json
app/Data/custom_thumbnail_presets.json → app/Storage/config/thumbnail-presets.json
app/Data/thumbnail_configs.json     → app/Storage/config/thumbnail-configs.json
app/Data/*_image_order.json         → app/Storage/config/image-orders.json (合并)
app/Config/*_config.php             → app/Config/modules/*.php
app/Config/*_sort.php               → app/Storage/config/sort-orders.json
app/storage/cache/                  → app/Storage/cache/
app/storage/logs/                   → app/Storage/logs/
```

## 🔧 标准化原则

### 1. **单一数据存储层**
- 所有数据文件统一放在 `app/Storage/`
- 区分 `data/`（业务数据）和 `config/`（配置数据）

### 2. **配置统一管理**
- 所有配置统一用PHP文件：`app/Config/`
- 按模块组织：`system/` 和 `modules/`

### 3. **模型职责清晰**
- 每个业务模块一个Model类
- 统一的数据访问接口

### 4. **命名规范**
- 文件名：kebab-case (中划线)
- 目录名：kebab-case
- 类名：PascalCase
